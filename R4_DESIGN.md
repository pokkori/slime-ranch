### 2. R2:31→R3:23 の8点低下の根本原因

R3で**SEを追加しただけ**で、根本的な問題を何も解決していない。スコアが下がった理由は「R2で加点された演出改善がR3で再評価された際に、それが表面的な改善に過ぎないことが明らかになった」ため。

**根本原因3つ:**

#### 原因1: 「戻ってくる理由」がない（放置ゲーム最大の欠陥）
- オフライン報酬は「コインが貯まる」だけ。コインで買えるものが装飾・背景・ブースターしかない
- 装飾はcoin_rate/offline_rate等のステータスボーナスだけで、**視覚的に牧場に反映されない**
- 「離れている間にスライムが成長した」「新しいスライムが出現した」等のサプライズがゼロ
- デイリーミッションは7つあるが、報酬がコインとジェムのみで、**特別なスライムや限定コンテンツがない**

#### 原因2: 合体が「作業」で「体験」ではない
- 同色・同ティアのスライムを2体ドラッグして重ねるだけの単純作業
- Merge Dragons!では「3マッチ」「5マッチでボーナス」「チェイン合体」があるが、本作は1:1合体のみ
- 合体結果が予測可能（green_1 + green_1 = green_2は確定）でサプライズがない
- **分裂（タップ）の戦略的意味が薄い**: タップで2体に分裂するが、同じ種になるだけ

#### 原因3: ビジュアルがCSS図形の域を出ない
- スライムがView+borderRadiusの円形。ティア装飾（角・翼・王冠）もCSS三角形
- Slime Rancherの3D表現やMerge Dragons!のアートワークと比較すると致命的に見劣り
- **牧場のフィールド自体が単色View + 絵文字（草・花・キノコ）** で世界観がない
- BGMがない（SEのみ）のは放置ゲームとして致命的。放置中に音楽がないと「生きた世界」を感じない

### 3. Merge Dragons! との決定的な差異（3つ）

| 項目 | Merge Dragons! | スライム牧場 |
|---|---|---|
| **合体ルール** | 3マッチ/5マッチコンボ、チェイン、ドラゴンが自動収穫 | 1:1ドラッグ合体のみ。自動要素ゼロ |
| **進行システム** | ステージ制+ワールドマップ+限定イベント+ドラゴンクエスト | フリーフィールド1面のみ。ステージもイベントもない |
| **コレクション報酬** | 図鑑完成でドラゴンパワー上昇→新エリア解放 | 図鑑は閲覧専用。コンプ報酬がキングスライムの説明文だけ（実際に獲得するロジックがない） |

## Part 2: R4改善設計（コードで解決可能な問題のみ）

### 改善1: オートスポーン＋放置サプライズシステム（戻ってくる理由の創出）

- **対象ファイル**: `src/engine/offline-reward.ts`, `src/store/gameStore.ts`, `src/components/OfflineRewardModal.tsx`
- **変更内容**:
  1. `offline-reward.ts`にオフラインスポーンロジックを追加: 離れていた時間に比例して新しいTier1スライムが自動出現（1時間につき1-2体、maxSlimes上限まで）
  2. 低確率（5%/時間）で「突然変異」が発生 — 通常の進化ツリーにない色違いスライム（例: `green_1` → `green_1_shiny`）がオフライン中に出現
  3. `OfflineRewardModal.tsx`に「留守中に起きたこと」セクション追加: 「新しいスライムが2体やってきました！」「突然変異が発生！虹色のみどりんが現れました！」
  4. `gameStore.ts`の`checkOfflineReward`を拡張し、スポーン結果をpendingOfflineRewardに含める
- **ユーザー行動の変化**: 「閉じて放置して、戻ったら何かが起きている」体験が生まれる。Idle Slayerや放置少女の「戻るとご褒美」ループの基盤
- **影響軸と点数変化**: 楽しい度 4→6（サプライズ要素）、差別化 3→4（突然変異は競合にない要素）

### 改善2: 3マッチ合体＋コンボチェインシステム

- **対象ファイル**: `src/engine/merge-logic.ts`, `src/store/gameStore.ts`, `app/(tabs)/ranch.tsx`
- **変更内容**:
  1. `merge-logic.ts`に`findMergeGroup(slimes, targetSlime, radius)`関数を追加: ドロップ地点の半径`MERGE_DISTANCE`以内に同色同ティアが3体以上ある場合、3マッチ合体を発動
  2. 3マッチ合体: 結果が1ティア上のスライム + ボーナスコイン（通常の3倍）
  3. 5マッチ合体: 結果が**2ティア上**のスライムに一気に進化 + 画面全体フラッシュ演出
  4. コンボチェイン: 合体結果のスライムが出現した位置に、さらに同色同ティアのスライムがいた場合、自動で連鎖合体（最大3連鎖）。連鎖ごとにコンボカウンター表示
  5. `ranch.tsx`のhandleDragEnd内で、まず3+マッチを検索→なければ通常の1:1合体にフォールバック
  6. コンボカウンターUI: 画面中央に大きく「COMBO x2!」「COMBO x3!」のアニメーションテキスト表示
- **ユーザー行動の変化**: 「同じスライムを3体以上近くに集めてからドロップ」という戦略が生まれる。5マッチの爽快感で「もっと合体したい」欲求が生まれる
- **影響軸と点数変化**: 楽しい度 4→7（コンボの快感）、使いやすさ 6→6（既存操作を壊さずフォールバック）、差別化 3→5（3マッチ+チェインはMerge系では珍しい）

### 改善3: マイルストーン進行システム（牧場ランク＋報酬）

- **対象ファイル**: `src/store/gameStore.ts`（state拡張）, `src/constants/milestones.ts`（新規）, `app/(tabs)/ranch.tsx`（ランク表示）, `src/components/MilestonePopup.tsx`（新規）
- **変更内容**:
  1. 新規 `src/constants/milestones.ts`: 牧場ランクシステムを定義
     - ランク1「はじまりの牧場」: 初回合体達成 → 報酬: コイン500
     - ランク2「見習い牧場主」: 図鑑5種発見 → 報酬: 新スロット解放
     - ランク3「一人前の牧場主」: Tier3スライム保有 → 報酬: 背景「深い森」無料解放
     - ランク4「凄腕牧場主」: 図鑑15種発見 → 報酬: 「オートマージ」30分無料
     - ランク5「伝説の牧場主」: Tier5スライム保有 → 報酬: 背景「天空庭園」無料解放
     - ランク6「スライム王」: 全36種コンプリート → 報酬: キングスライム実際に出現
  2. 新規 `src/components/MilestonePopup.tsx`: ランクアップ時のフルスクリーン演出（金色のフレーム、新ランク名表示、報酬アニメーション）
  3. `gameStore.ts`に`ranchRank`と`checkMilestone()`を追加。合体・発見のたびにチェック
  4. `ranch.tsx`のヘッダーに現在のランクバッジ + 次ランクまでの進捗バー表示
- **ユーザー行動の変化**: 「次のランクまであと2種！」という目標ができる。ランクアップの報酬が実際に有用（スロット解放・背景・ブースター）なので、達成欲がドライブする
- **影響軸と点数変化**: 楽しい度 6→7（目標と報酬の明確化）、使いやすさ 6→7（進捗の可視化）

### 改善4: BGMシステム（放置ゲームの「世界に居る感覚」）

- **対象ファイル**: `src/utils/sound.ts`（BGM関数追加）, `app/(tabs)/ranch.tsx`（BGM再生制御）, `app/(tabs)/settings.tsx`（BGM音量スライダー）
- **変更内容**:
  1. `sound.ts`にWeb Audio APIベースのプロシージャルBGM生成を追加:
     - ペンタトニックスケールのランダムメロディ（C-D-E-G-A）を4小節ループ
     - 背景テーマごとにキー変更: meadow=C major, forest=A minor, beach=F major, volcano=D minor
     - 低音のベースドローン（オシレーターの低周波サイン波）を常時再生してアンビエント感を出す
  2. `ranch.tsx`でアプリフォアグラウンド時にBGM開始、バックグラウンド時に停止
  3. `settings.tsx`のBGM音量スライダー連動（既存のbgmVolume設定を活用）
- **ユーザー行動の変化**: 音楽があることで「牧場に居る」感覚が生まれる。放置中に戻ってきたとき、音楽が聞こえることで「あの世界に帰ってきた」感覚
- **影響軸と点数変化**: 表現性 4→6（音楽で世界観が立つ）

### 改善5: Canvas OGPシェアカード（バズり度の根本解決）

- **対象ファイル**: `src/utils/share-card.ts`（新規）, `app/(tabs)/ranch.tsx`（シェアボタン追加）, `app/(tabs)/encyclopedia.tsx`（シェア機能強化）
- **変更内容**:
  1. 新規 `src/utils/share-card.ts`: HTML Canvas APIでOGPカード（1200x630）を生成
     - 牧場のスクリーンショット風: 背景テーマの色 + 所持スライムの描画（円形+目+口をCanvasに再現）
     - テキスト情報: 「スライム牧場 | ランクX: [ランク名] | 図鑑 XX/38種 | 最高ティア: [名前]」
     - ハッシュタグ: 「#スライム牧場 #放置ゲーム」
  2. `ranch.tsx`のヘッダーにカメラアイコンのシェアボタン追加。タップでCanvas生成 → data URL → navigator.share() or clipboard
  3. `encyclopedia.tsx`のシェア機能を強化: テキストだけでなくCanvasカード生成
- **ユーザー行動の変化**: ビジュアル付きシェアが可能になり、SNSでの拡散が視覚的にインパクトを持つ
- **影響軸と点数変化**: バズり度 3→5（OGP画像があるとSNSのCTR 2-3倍）

### 改善6: アプリ内スライム自動生成（放置の本質）

- **対象ファイル**: `src/store/gameStore.ts`, `app/(tabs)/ranch.tsx`
- **変更内容**:
  1. `gameStore.ts`に`autoSpawnTimer`ロジック追加: 30秒ごとに牧場にランダムTier1スライムが1体自動出現（maxSlimes上限まで）
  2. 自動出現時にフィールド上部からふわふわ降りてくるアニメーション（既存のisNew + scaleアニメーションを活用）
  3. `ranch.tsx`のphysicsループ内に自動スポーンチェックを組み込み
  4. ショップのデコレーション「虹のアーチ」「風鈴」のspawn_rateボーナスが実際にスポーン間隔を短縮するように実装（現在は定義のみで未反映）
- **ユーザー行動の変化**: 「放置していれば牧場にスライムが増える」→「増えたスライムを整理して合体」→「高ティアスライムができる」→「コインが増える」→「ショップで強化」の基本ループが完成する
- **影響軸と点数変化**: 楽しい度 7→8（放置ゲームの基本ループ完成）、差別化 5→5（放置ゲームでは標準的だが現在ゼロなので大幅改善）

### コードで解決できない問題（正直に申告）
1. **収益性（2/10）**: App Store / Google Play への公開とIAP実装が必要。KOMOJU/Stripeの審査はコード外
2. **SEO・集客（1/10）**: Expo Web exportしてもインデックス不可能。ASO（アプリストア最適化）はストア公開が前提
3. **表現性の上限**: CSS View描画の限界は根本的にある。SVGやLottie、もしくはExpo GLでの描画に移行しない限り、表現性7以上は困難
4. **ゲームコアループの構造限界**: にゃんこ大戦争やSlime Rancherとの本質的な差は「ステージ設計」「ナラティブ」「手触り」であり、CSSアートの放置マージゲームで8点（競合）水準に到達するのは非常に困難
